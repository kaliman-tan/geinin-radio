# 芸人ラジオまとめアプリ 要件定義書

> この文書はVibe Coding（Cursor / Windsurf / Replit等）への指示書としてそのまま使用できます。

---

## 1. サービス概要

**サービス名（仮）：** 芸ラジ（Geiradio）

**一言説明：**  
芸人を選ぶと、その芸人がラジオ・Podcastをやっている全媒体へのリンクが一覧で見られるWebアプリ。

**解決する課題：**  
芸人ラジオは YouTube・Spotify・Apple Podcast・GERA・radiko など複数の媒体に分散しており、ファンが追いかけるのが大変。

**方針：**
- 音声の直接再生は行わない（著作権・規約上の安全ライン）
- 各媒体の外部リンクへ遷移する形式
- 芸人の写真・サムネイルは使用しない（パブリシティ権対策）
- テキスト情報とバッジ表示のみで構成する

---

## 2. 技術スタック

| 項目 | 採用技術 | 理由 |
|------|----------|------|
| フレームワーク | Next.js (App Router) | Vercelへの無料デプロイが容易 |
| スタイリング | Tailwind CSS | Vibe Codingと相性が良い |
| データ管理 | JSONファイル（初期）| DBなしで始められる |
| ホスティング | Vercel（無料枠） | GitHubと連携して即公開可能 |
| 言語 | TypeScript | |

> **Vibe Coding補足：** 上記構成で `npx create-next-app@latest` から始めてください。

---

## 3. 画面構成

### 画面① ホーム（芸人一覧）

**URL：** `/`

**表示内容：**
- ヘッダー：サービス名 + ベルアイコン（将来の通知用）
- 検索バー：芸人名のインクリメンタルサーチ
- 芸人リスト（各行に下記を表示）
  - イニシャルアバター（写真なし）
  - 芸人名
  - 番組数 + 媒体一覧テキスト
  - 媒体バッジ（YT / SP / AP / GE / RK）
- タブバー：ホーム / お気に入り / 検索

### 画面② 芸人詳細

**URL：** `/artist/[id]`

**表示内容：**
- 戻るボタン
- 芸人名・メンバー名
- 番組リスト（各行に下記を表示）
  - 媒体アイコン（カラーバッジ）
  - 番組名
  - 媒体名 + 更新頻度
  - 「開く」ボタン → 外部リンク（新しいタブ）
- お気に入り追加ボタン（ローカルストレージで管理）

### 画面③ お気に入り

**URL：** `/favorites`

**表示内容：**
- お気に入り登録した芸人の一覧（画面①と同形式）
- 未登録時：空のステートメッセージ

## 4. データ構造

### 芸人データ（`/data/artists.json`）

```json
[
  {
    "id": "shimofuri-myojo",
    "name": "霜降り明星",
    "members": ["せいや", "粗品"],
    "programs": [
      {
        "title": "霜降りチャンネル",
        "platform": "youtube",
        "url": "https://www.youtube.com/@shimofuri",
        "frequency": "毎週更新"
      },
      {
        "title": "霜降りラジオ",
        "platform": "spotify",
        "url": "https://open.spotify.com/show/...",
        "frequency": "週1回"
      },
      {
        "title": "霜降りGERA",
        "platform": "gera",
        "url": "https://gera.fan/...",
        "frequency": "不定期"
      }
    ]
  }
]
```

### 媒体の種類（`platform` フィールドの値）

| 値 | 表示名 | バッジ色 |
|----|--------|----------|
| `youtube` | YouTube | 赤 |
| `spotify` | Spotify | 緑 |
| `apple_podcast` | Apple Podcast | 紫 |
| `gera` | GERA | オレンジ |
| `radiko` | radiko | 青 |
| `other` | その他 | グレー |

## 5. UIデザイン仕様

- **フォント：** システムフォント（sans-serif）
- **配色：** 白背景・グレー基調のミニマルデザイン
- **写真：** 一切使用しない。芸人はイニシャルのアバター円で表示
- **媒体バッジ：** 小さいカラーピル（上記の色に準拠）
- **リンク：** すべて `target="_blank" rel="noopener noreferrer"` で外部遷移
- **レスポンシブ：** スマートフォン優先（max-width: 480px を基準）

## 7. やってはいけないこと（開発上の制約）

| 禁止事項 | 理由 |
|----------|------|
| 芸人の顔写真・サムネイルの表示 | パブリシティ権・著作権侵害のリスク |
| 音声・動画の直接再生 | 各媒体の利用規約違反 |
| radikoのスクレイピング | 利用規約で明示的に禁止 |
| GERAのAPIへの無断アクセス | 公開APIなし・規約違反 |
| YouTubeの音声のみ抽出再生 | YouTube利用規約違反 |

## 9. Vibe Codingへの指示文（コピペ用）

以下をそのままCursor/Windsurfのチャットに貼り付けて開発を開始できます。

```
以下の仕様でWebアプリを作ってください。

- フレームワーク: Next.js (App Router) + TypeScript + Tailwind CSS
- 芸人ごとにラジオ番組リンクをまとめるサービス
- 音声の再生機能は不要。外部リンクへの遷移のみ
- データは /data/artists.json のJSONファイルで管理
- 写真は使わない。イニシャルのアバター（円）で代替
- 媒体（YouTube/Spotify/Apple Podcast/GERA/radiko）をカラーバッジで表示
- スマートフォン優先のレスポンシブデザイン
- シンプル・ミニマルなUIで、見やすさ最優先

まず以下の2画面を作成してください：
1. 芸人一覧ページ（/）：検索バー付き、芸人リスト表示
2. 芸人詳細ページ（/artist/[id]）：番組リストと外部リンク表示

サンプルデータとして霜降り明星・空気階段・オードリーの3名分を
artists.json に含めてください。
```
